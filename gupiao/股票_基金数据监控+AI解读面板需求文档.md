# 股票/基金数据监控+AI解读面板需求文档

# 1. 文档概述

## 1.1 文档目的

本文档用于明确「股票/基金数据监控+AI解读面板」项目的全部需求，包括功能需求、非功能需求、合规要求等，作为开发、测试、验收的核心依据，确保项目开发方向一致，贴合用户实际需求，同时适配 Claude 辅助开发的流程，降低开发难度。

## 1.2 项目定位

本项目是一款面向普通个人投资者的轻量化金融数据工具，核心价值是「简化复杂金融数据，用 AI 提供通俗解读」，聚焦“数据监控+AI 分析”两大核心，不涉及任何交易功能，规避合规风险，区别于传统专业金融软件的复杂操作，主打简洁、易用、易懂，让普通投资者能快速掌握自选标的动态和核心信息。

## 1.3 目标用户

核心用户：普通个人投资者（学生、上班族、中老年投资者等），具备基础投资认知，无需专业金融知识，需要便捷监控持仓/观察标的，希望获取通俗的数据分析和解读，无需复杂操作。

次要用户：初级金融从业者，需要快速汇总标的数据、生成简易分析报告，提升工作效率。

## 1.4 开发环境适配

后端：Python（支持 Flask/FastAPI，适配 Claude API 调用、金融数据 API 对接）；前端：支持 ECharts/Chart.js 数据可视化，适配 PC 端、移动端响应式；数据存储：轻量数据库（SQLite/MySQL）+ 可选向量库（用于优化 AI 解读效率）；AI 支持：对接 Claude API，实现金融数据解读、新闻摘要等功能。

# 2. 核心需求总览

本项目核心满足用户“监控自选标的、看懂金融数据、获取精准解读”三大需求，拆解为 4 大核心模块，按开发优先级排序：

1. 自选管理模块：实现股票/基金的添加、分组、删除、编辑，支持多标的管理；

2. 数据监控模块：展示实时/准实时数据、历史走势可视化，设置价格预警；

3. AI 解读模块：对接 Claude API，实现单标的解读、组合解读、财经新闻摘要；

4. 辅助功能模块：收益计算、数据导出、用户系统（可选）、主题切换。

# 3. 详细功能需求

## 3.1 自选管理模块（优先级：最高）

### 3.1.1 标的添加

- 支持手动输入股票代码（A股/港股/美股）、基金代码，自动匹配标的名称、所属板块（股票）、基金类型（货币/债券/股票型等），避免输入错误；

- 支持模糊搜索：输入标的名称/简称，可搜索匹配相关标的，选择后添加至自选；

- 添加时可选择分组（如“持仓股”“观察股”“基金组合”），支持自定义分组名称；

- 添加成功后，自动同步该标的的基础数据（现价/净值、涨跌幅等），无需手动刷新。

### 3.1.2 标的管理

- 支持单个/批量删除自选标的，删除前弹出确认提示，避免误操作；

- 支持标的分组调整：可将标的从一个分组移动至另一个分组；

- 支持自定义标的排序（按涨跌幅、名称、添加时间排序）；

- 显示标的基础信息：股票（代码、名称、所属板块）、基金（代码、名称、基金公司、基金类型）。

### 3.1.3 分组管理

- 支持添加、编辑、删除自定义分组，分组名称不重复；

- 支持按分组查看标的，切换分组时，快速刷新该分组下的所有标的数据；

- 默认分组：“未分组”，新添加未选择分组的标的自动归入该分组。

## 3.2 数据监控模块（优先级：最高）

### 3.2.1 基础数据展示

#### 3.2.1.1 股票数据

展示核心数据，无冗余信息，通俗标注：

- 核心指标：现价、涨跌幅（红色涨、绿色跌，标注具体百分比）、成交量（标注单位：万/亿）、换手率；

- 辅助指标：开盘价、收盘价（昨日）、最高价、最低价；

- 数据更新频率：准实时（5分钟刷新一次，可手动触发强制刷新），标注最后刷新时间。

#### 3.2.1.2 基金数据

聚焦基金投资者核心关注数据：

- 核心指标：单位净值、累计净值、当日估算净值（基金未收盘时）、估算涨跌幅；

- 辅助指标：近1周、近1月、近3月、近1年收益率（标注百分比，红色涨、绿色跌）；

- 数据更新频率：基金净值每日更新（同步官方收盘后数据），估算净值每10分钟刷新一次。

### 3.2.2 历史数据可视化

- 支持单个标的历史走势查看，可选择时间周期：近7天、近30天、近90天、近1年；

- 图表类型：折线图（核心）、柱状图（成交量/净值变化），图表简洁，标注关键价格/净值点位；

- 交互功能：鼠标悬浮在图表上，显示对应日期的具体数据（现价/净值、涨跌幅）；支持缩放、拖拽查看细节；

- 组合走势：支持选择同一分组下的多个标的，对比展示历史走势（同一张图表，不同颜色区分）。

### 3.2.3 价格预警

- 支持为单个标的设置预警规则：股票（股价上限、股价下限）、基金（净值上限、净值下限）；

- 预警方式：站内通知（页面右上角弹窗提示）、邮件通知（用户可选择是否开启，需填写邮箱并验证）；

- 预警管理：查看所有预警规则，支持编辑、删除预警，标记已读预警；

- 触发逻辑：数据刷新时，若标的价格/净值达到预警阈值，立即触发预警，同一预警规则1小时内不重复触发。

## 3.3 AI 解读模块（优先级：中高）

### 3.3.1 单标的 AI 解读

#### 3.3.1.1 股票解读

对接 Claude API，输出通俗解读（无专业术语，控制在300字以内），包含3个核心部分：

- 当日涨跌原因：结合公开财经新闻、行业动态，通俗说明（如“今日上涨主要因行业政策利好，板块整体走强”）；

- 短期趋势判断：基于近7天走势和市场环境，给出中性解读（如“短期走势偏震荡，需关注明日行业数据发布”）；

- 风险提示：通用风险提醒（如“市场有波动，投资需谨慎，本解读仅做数据参考”），不做具体投资建议。

#### 3.3.1.2 基金解读

聚焦基金投资者需求，解读内容简洁易懂（300字以内）：

- 净值波动解读：说明当日/近期净值波动原因（如“今日净值下跌，主要因持仓股票板块回调”）；

- 持仓与风格分析：简要说明基金持仓结构（如“持仓以消费、科技板块为主，偏向成长风格”）；

- 持有建议：中性提示（如“适合长期持有，短期波动属正常现象，建议结合自身风险承受能力调整持仓”），不做具体买卖建议。

### 3.3.2 组合 AI 解读

- 对用户自选分组内的所有标的进行整体分析，输出组合解读（400字以内）；

- 核心内容：组合整体收益情况、板块分布占比、当日涨跌核心驱动因素、组合风险提示；

- 示例：“你的持仓组合今日整体收益+1.8%，其中消费类股票贡献主要收益，占比60%；科技类股票小幅回调，占比30%。今日上涨主要因消费板块政策利好，短期组合走势偏乐观，需关注后续板块轮动风险。”

### 3.3.3 财经新闻 AI 摘要

- 自动抓取热门财经新闻、行业资讯（来源为公开合规平台），AI 提炼核心要点（每篇100字以内）；

- 关联匹配：将新闻与用户自选标的关联，标注“关联标的”，方便用户快速查看与自身持仓相关的资讯；

- 新闻展示：按发布时间排序，支持筛选（按行业、关联标的筛选），点击可查看新闻原文（跳转至原链接）；

- 更新频率：每日更新3-5条核心新闻，避免信息过载。

### 3.3.4 AI 解读设置

- 支持调整解读风格：通俗版（默认）、简洁版（150字以内）、稍专业版（适合初级从业者）；

- 支持手动触发解读：点击“重新解读”，可获取最新的 AI 解读内容；

- 解读历史：保存近7天的 AI 解读记录，支持查看历史解读。

## 3.4 辅助功能模块（优先级：中）

### 3.4.1 收益计算器

- 股票收益计算：输入持仓成本、持仓数量，自动计算浮盈/浮亏、收益率（收益率=（现价-成本）/成本×100%）；

- 基金收益计算：输入买入金额、买入净值、当前净值，自动计算浮盈/浮亏、收益率；支持分批买入，输入多笔买入记录，自动计算平均成本和总收益；

- 结果展示：清晰标注计算结果，支持手动修改参数（成本、数量等），实时更新计算结果。

### 3.4.2 数据导出

- 支持导出自选标的基础数据（代码、名称、现价/净值、涨跌幅等），导出格式为 Excel/CSV；

- 支持导出单个标的历史数据（按选择的时间周期），包含日期、价格/净值、涨跌幅；

- 支持导出收益计算结果，方便用户存档、后续分析。

### 3.4.3 用户系统（可选）

- 基础功能：注册（邮箱/手机号）、登录（账号密码/验证码）、密码找回；

- 数据同步：登录后，自选列表、分组、预警设置、解读历史、收益记录自动同步，跨设备使用；

- 个人中心：修改个人信息（昵称、密码、邮箱），查看登录记录，管理邮件通知设置。

### 3.4.4 界面与交互优化

- 主题切换：支持浅色模式（默认）、暗黑模式，切换后整体界面同步适配；

- 响应式设计：适配 PC 端、移动端（手机、平板），界面布局自动调整，保证使用体验；

- 搜索功能：全局搜索，可搜索标的代码、名称、分组名称，快速定位；

- 加载提示：数据加载、AI 解读生成时，显示加载动画，避免用户误以为页面卡顿。

# 4. 非功能需求

## 4.1 性能需求

- 页面加载速度：首页加载时间≤3秒，标的数据刷新时间≤1秒，AI 解读生成时间≤5秒；

- 并发能力：支持同时在线100人以内，数据请求无明显延迟；

- 稳定性：连续运行72小时无崩溃，数据抓取、AI 调用无异常，预警触发准确。

## 4.2 易用性需求

- 操作简洁：核心功能（添加标的、查看数据、查看解读）操作步骤≤3步；

- 界面友好：无冗余信息，按钮、图表布局合理，文字清晰，颜色搭配舒适（避免刺眼颜色）；

- 容错性：用户输入错误（如无效代码）时，给出明确提示（如“该代码不存在，请重新输入”），不崩溃；

- 帮助提示：核心功能模块（如预警设置、收益计算）提供简单帮助文案，指导用户操作。

## 4.3 安全性需求

- 数据安全：用户自选列表、收益记录、个人信息（若有）加密存储，不泄露；

- API 安全：对接金融数据 API、Claude API 时，密钥加密存储，不暴露在前端；

- 防恶意请求：限制单个 IP 每日请求次数，避免恶意刷新、爬取数据。

## 4.4 兼容性需求

- 浏览器兼容：支持 Chrome、Edge、Firefox 最新版本，以及 Safari 14.0+ 版本；

- 设备兼容：PC 端（Windows、Mac）、移动端（iOS、Android），屏幕尺寸适配（最小320px宽度）；

- 系统兼容：后端支持 Windows、Linux 系统，可正常部署运行。

# 5. 合规要求（重点）

- 免责声明：页面显著位置（首页底部、AI 解读页面顶部）标注免责声明，明确“本工具仅提供金融数据参考和 AI 解读，不构成任何投资建议，投资者据此操作风险自担”；

- 数据来源：仅使用公开、合规的金融数据 API（如 tushare、akshare、新浪财经公开接口），不爬取付费数据、违规数据，标注数据来源；

- AI 解读规范：AI 解读内容仅做数据总结和分析，不包含“买入”“卖出”“持有”等具体投资建议，不预测股价/净值走势；

- 版权声明：若使用第三方图标、框架，需符合版权要求，标注版权信息；财经新闻仅做摘要，跳转至原链接，不侵权。
