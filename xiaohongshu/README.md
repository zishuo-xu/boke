# 小红书博主视频早期流量追踪工具

专为小红书博主设计的轻量化视频早期流量监控工具，支持记录并可视化展示视频发布后1小时、4小时、8小时、24小时的关键数据，通过流量曲线直观呈现早期流量增长趋势。

## 功能特性

### 博主管理
- 新增博主：手动录入博主昵称、账号链接、备注
- 编辑博主：修改博主基础信息
- 删除博主：删除博主（带二次确认）
- 博主切换：顶部下拉选择器，一键切换不同博主

### 视频管理
- 新增视频：录入视频标题、链接、发布时间、标签
- 视频筛选：按时间段、流量表现（爆款/低流量）筛选
- 视频搜索：通过标题或链接关键词搜索
- 编辑/删除视频：修改或删除视频信息

### 数据录入
- 分段录入：按时间节点（1h/4h/8h必录，24h可选）录入流量数据
- 数据指标：播放量、点赞数、收藏数、评论数、完播率
- 数据验证：自动验证数据有效性
- 补录功能：支持后续补录或修改已有数据

### 数据展示
- 数据卡片：清晰展示视频基础信息和各时间节点完整数据
- 流量曲线：自动生成折线图，可切换查看不同指标
- 增长分析：显示各指标从首节点到末节点的增长率

### 数据导出
- 导出范围：支持导出当前博主所有视频或选中视频
- 导出格式：Excel、CSV两种格式
- 导出内容：包含视频基础信息及所有时间节点的完整数据

## 快速开始

### 环境要求
- Python 3.8 或更高版本
- pip（Python包管理器）

### 安装步骤

1. 克隆或下载项目到本地

```bash
cd xiaohongshu
```

2. 安装依赖包

```bash
pip install -r requirements.txt
```

3. 运行应用

```bash
streamlit run app.py
```

4. 应用将在浏览器中自动打开（默认 http://localhost:8501）

## 使用指南

### 1. 添加博主

首次使用时，请先在左侧侧边栏添加博主：
- 输入博主昵称（必填）
- 输入小红书账号链接（可选）
- 输入备注（可选）
- 点击"添加"按钮

### 2. 新增视频

1. 选择当前博主
2. 点击"新增视频"标签
3. 录入视频信息：
   - 视频标题（必填）
   - 视频链接（必填）
   - 发布时间（必填）
   - 标签（可选）
4. 点击"添加视频"

### 3. 录入流量数据

1. 在视频列表中，点击"录入数据"按钮
2. 选择时间节点（1h/4h/8h为必录节点，24h为可选）
3. 录入流量数据：
   - 播放量（必填）
   - 点赞数（必填）
   - 收藏数（必填）
   - 评论数（必填）
   - 完播率（必填，百分比）
4. 点击"保存数据"

### 4. 查看视频详情

1. 在视频列表中，点击"查看详情"按钮
2. 查看视频数据卡片和流量曲线
3. 可选择要显示的指标（支持多选）
4. 查看增长分析

### 5. 筛选视频

在"视频列表"标签中，选择筛选方式：
- **全部**：显示所有视频
- **按时间**：选择起止日期，按发布时间筛选
- **按流量表现**：筛选爆款或低流量视频（可自定义阈值）
- **搜索**：输入关键词，按标题或链接搜索

### 6. 导出数据

1. 点击"导出数据"标签
2. 选择导出范围：当前博主所有视频 / 选中视频
3. 选择导出格式：Excel / CSV
4. 点击"导出数据"
5. 下载生成的文件

## 技术栈

- **Web框架**：Streamlit 1.38+
- **数据库**：SQLite（本地轻量级存储）
- **数据处理**：pandas
- **可视化**：Streamlit原生图表（基于Plotly）
- **数据导出**：openpyxl（Excel）、pandas（CSV）

## 数据存储

所有数据存储在本地SQLite数据库文件 `xiaohongshu.db` 中，包括：
- 博主信息
- 视频信息
- 流量数据

数据文件位于项目根目录下，可随时备份。

## 注意事项

1. **数据安全**：所有数据存储在本地，请定期备份数据库文件
2. **删除确认**：删除博主或视频前会弹出确认提示，请谨慎操作
3. **数据验证**：录入数据时会自动验证有效性，请确保与小红书后台数据一致
4. **必录数据**：1h/4h/8h三个时间节点的数据为必录，24h为可选
5. **浏览器兼容**：推荐使用Chrome、Edge、Firefox最新版本或Safari 14.0+

## 常见问题

**Q: 应用无法启动？**
A: 请确保已安装所有依赖包：`pip install -r requirements.txt`

**Q: 数据会丢失吗？**
A: 数据存储在本地SQLite文件中，只要不删除 `xiaohongshu.db` 文件，数据不会丢失。建议定期备份。

**Q: 如何更换数据存储位置？**
A: 可以修改 `models/database.py` 中的 `get_db_path()` 函数，指定自定义路径。

**Q: 支持多人同时使用吗？**
A: 当前版本为单用户版本，数据存储在本地。如需多人协作，需要部署到服务器并添加用户认证功能。
