# AI狼人杀项目续接文档（DEV HANDOFF）

更新时间：2026-03-01

## 1. 当前项目状态

- 项目已从纯前端改为前后端架构。
- 后端：Python Flask（`backend/app.py`）
- 前端：`index.html` + `styles.css` + `app.js`
- 通信方式：
  - 控制与配置：HTTP API (`/api/*`)
  - 实时状态：SSE (`/api/events`)

## 2. 已实现能力

### 2.1 游戏流程
- 6-12 人配置与角色分配。
- 自动流程：夜间 -> 天亮 -> 发言 -> 投票 -> 胜负判定。
- 主持人引导已加入，夜间按固定环节推进：
  - 狼人环节
  - 预言家环节
  - 女巫环节
  - 守卫环节
  - 夜间结算
- 角色出局时，对应环节会由主持人明确“跳过”。

### 2.2 AI与思考
- 角色逻辑：狼人、村民、预言家、女巫、猎人、守卫。
- 思考来源展示：
  - Mock 时显示 `虚拟(...)`
  - LLM 成功时显示模型名（如 `GLM-5`）
- 思考面板支持实时显示“模型思考中”状态。

### 2.3 模型配置
- 支持 `mock / openai / glm`。
- 配置保存时会做可用性校验并返回提示。
- LLM 配置持久化：`backend/llm_config.json`
- 当前为单模型策略：夜间/发言/投票都使用同一模型（已回退掉分阶段模型）。

### 2.4 前端交互
- 三栏布局：玩家列表 / 主场景 / 思考面板。
- 顶部支持开始、暂停、继续、重置、速度设置。
- “显示/隐藏身份”按钮切换左侧角色显隐。
- 模型配置区域有成功/失败提示条。

## 3. 关键技术决策（最近）

1. 从轮询改为 SSE 实时推送（前端不再 800ms 轮询）。
2. 模型调用改为默认非流式（确认后执行）：
   - 原因：当前链路流式稳定性较差，容易“无增量/超时”。
   - 当前策略：一次性非流式请求，失败才回退 mock。
3. 模型等待时间统一放宽到最长 60 秒。

## 6. 已知问题 / 风险

1. LLM稳定性依赖外部服务
- 即便非流式，仍可能出现超时或空内容，系统会回退 `虚拟(...)`。

2. 规则真实性仍可继续打磨
- 目前为可运行、可观察版本，不是赛事级严谨判定器。

3. 单房间内存态
- 对局数据不持久化（刷新后状态依赖后端进程）。
- LLM配置持久化，但对局本身不持久化。

### 常见问题

- `ModuleNotFoundError: flask`
  - 说明依赖没装到当前 `.venv`。
- 模型一直显示 `虚拟(...)`
  - 看来源后缀，通常是超时、返回空或接口参数问题。
- 页面不更新
  - 检查 `/api/events` 是否可连通（SSE）。
