# AI机器人狼人杀Web项目需求文档

# 1. 文档概述

## 1.1 文档目的

本文档明确AI机器人狼人杀Web项目的核心需求、功能范围、界面要求、非功能需求及约束条件，为项目设计、开发、测试及验收提供统一标准和依据，确保项目交付符合预期目标。

## 1.2 项目背景

本项目旨在开发一个纯可视化Web网站，实现多个AI机器人自主进行狼人杀游戏，用户可通过网页实时观察AI机器人的每一步操作（如杀人、验人、投票等），并清晰查看每个AI机器人的完整思考过程，直观了解AI的决策逻辑，打造沉浸式、可观察的AI博弈展示场景。

## 项目目标

- 实现AI机器人自主完成狼人杀全流程游戏，无需人类参与干预。

- Web页面清晰可视化游戏进程、AI角色状态及每一步操作。

- 完整展示每个AI机器人的思考过程，包括信息分析、决策依据。

- 保证游戏逻辑严谨、AI决策合理，界面流畅、交互友好。

## 适用范围

本文档适用于本项目的产品设计、前端开发、后端开发、测试人员及项目管理人员，作为各环节工作的核心依据。

# 2. 核心需求

本项目核心需求为“AI机器人自主博弈+操作与思考过程可视化”，具体拆解为以下3点：

1. AI机器人需求：多个独立AI机器人，需接入LLM模型，且支持LLM模型动态配置（可切换不同模型、调整模型参数）；AI基于配置的模型能力结合对应角色的决策逻辑，自主完成狼人杀全流程行动，思考过程、发言内容由LLM模型生成，确保逻辑连贯、贴合角色立场。

2. 游戏流程需求：严格遵循狼人杀标准规则，实现完整游戏流程，可自动推进、重置游戏。

3. 可视化需求：Web页面直观展示游戏状态、AI操作及思考过程，支持清晰查看、回溯。

# 3. 详细功能需求

## 3.1 游戏基础配置功能

- 角色配置：支持6-12人狼人杀配置，可自动分配角色（狼人、村民、预言家、女巫、猎人等），角色数量符合标准规则（如12人局：4狼人、4平民、1预言家、1女巫、1猎人、1守卫）。

- AI数量配置：支持用户设置AI机器人数量（6-12个），每个AI对应一个唯一身份（名称、头像可自定义或系统默认）。

- 游戏重置：支持一键重置游戏，重新分配角色、清空游戏记录，开始新一局游戏。

- 游戏速度控制：支持调节游戏进程速度（快、中、慢），可暂停游戏，便于查看细节。

## AI机器人功能

### 3.2.1 角色逻辑需求

每个AI机器人根据自身角色，具备独立的决策逻辑，行动符合角色规则，同时生成对应的思考过程：

- 狼人：夜间自主选择袭击目标，白天伪装身份、引导投票，思考过程需包含“身份识别、队友判断、目标选择依据”。

- 预言家：夜间自主选择查验目标，白天可选择是否跳身份、公布查验结果，思考过程需包含“查验目标选择、身份判断、发言策略”。

- 女巫：夜间可选择救人或毒人（遵循女巫规则：解药、毒药各用一次，不可同时使用），思考过程需包含“判断被刀目标身份、毒人目标选择依据”。

- 猎人：被投票出局或被刀时，可选择开枪带走一名玩家，思考过程需包含“判断自身处境、开枪目标选择依据”。

- 村民：无特殊技能，白天根据发言、投票信息判断身份，参与投票，思考过程需包含“身份分析、投票目标选择依据”。

### 3.2.2 思考过程需求

- 实时生成：AI每执行一次操作（夜间行动、白天发言、投票）前，自动生成结构化思考过程，同步展示在Web页面。

- 内容要求：思考过程需逻辑连贯，包含“已知信息（存活状态、发言记录、操作记录）、自身角色定位、决策分析、最终行动决定”，语言通俗易懂，模拟人类思考逻辑。

- 追溯查看：支持点击AI机器人，查看其本局游戏所有思考记录，按时间顺序排列。

### 3.2.3 发言功能

- 白天发言阶段，AI机器人按随机顺序依次发言，发言内容符合自身角色立场，与思考过程一致。

- 发言内容可在页面实时展示，支持回放发言记录。

## 3.3 游戏流程功能

严格遵循狼人杀标准流程，自动推进，无需人工干预，流程如下：

1. 游戏开始：分配角色，AI机器人查看自身角色，生成初始思考（确认自身身份、游戏目标）。

2. 夜间阶段：按“狼人→预言家→女巫→猎人（仅被刀时）”的顺序，AI依次执行操作，同步展示操作行为及思考过程。

3. 天亮阶段：公布夜间死亡信息（若有），展示死亡玩家身份（可选，可设置“死后揭示身份”或“游戏结束后统一揭示”）。

4. 发言阶段：存活AI按顺序发言，展示发言内容及思考过程。

5. 投票阶段：存活AI自主投票，投票结束后公布投票结果，出局玩家展示（若有），生成出局动画及AI相关思考。

6. 胜负判定：每轮结束后，判断是否满足胜负条件，若满足则结束游戏，公布胜负结果及所有玩家身份；若不满足则进入下一轮。

## 3.4 前端可视化功能

### 3.4.1 页面布局需求

- 顶部区域：显示游戏当前阶段（夜间/天亮/发言/投票/游戏结束）、游戏速度控制按钮、重置游戏按钮。

- 左侧区域：展示所有AI机器人头像、名称、角色标识（游戏结束前可隐藏，仅AI自身可见，用户可选择是否查看所有角色）、存活状态（存活/死亡，用不同颜色或图标区分）。

- 中间区域：游戏主场景，展示夜间操作动画（如狼人刀人、预言家验人）、天亮死亡信息、发言内容、投票界面及结果。

- 右侧区域：AI思考过程展示面板，实时显示当前行动AI的思考过程，支持切换查看不同AI的思考记录，可折叠/展开。

### 3.4.2 交互需求

- 角色查看：点击AI机器人头像，可查看其当前状态（存活/死亡）、已执行操作、思考记录，游戏结束后可查看其真实角色。

- 思考过程交互：思考过程支持逐字打字机效果（增强沉浸感），可暂停、回放，支持复制思考内容。

- 游戏控制：支持暂停/继续游戏、调节游戏速度、一键重置，操作便捷，反馈及时。

- 记录查看：游戏结束后，支持查看本局完整游戏记录，包括每一轮操作、发言、思考过程、投票结果。

### 3.4.3 视觉反馈需求

- 状态区分：用不同颜色区分AI存活/死亡状态（如存活为绿色，死亡为灰色），不同角色用不同图标或颜色标识。

- 操作动画：夜间操作、投票、出局等环节，添加简单动画效果，增强可视化体验（如狼人刀人动画、投票计数动画）。

- 提示信息：游戏阶段切换、AI操作、胜负结果等，添加明确提示信息，便于用户理解。

## 3.5 胜负判定功能

- 狼人胜利条件：所有村民或所有神职人员（预言家、女巫、猎人等）被淘汰。

- 好人胜利条件：所有狼人被淘汰。

- 游戏结束后，自动公布胜负结果、所有AI机器人的真实角色，展示本局游戏核心统计（如淘汰顺序、投票次数等）。

# 4. 非功能需求

## 4.1 性能需求

- 页面加载速度：首次加载时间≤3秒，游戏过程中切换阶段、展示思考过程无卡顿（延迟≤500ms）。

- AI响应速度：AI执行操作、生成思考过程的响应时间≤1秒，不影响游戏流程推进。

- 并发支持：支持单用户多窗口查看，或多用户同时访问，无崩溃、无数据错乱。

## 4.2 兼容性需求

- 浏览器兼容：支持主流浏览器（Chrome、Edge、Firefox、Safari），最新版本及前两个版本均可正常显示和操作。

- 设备兼容：支持电脑端（台式机、笔记本），适配不同屏幕分辨率（最小分辨率≥1366×768），无需适配移动端。

## 4.3 易用性需求

- 界面简洁清晰，操作逻辑简单，无需用户学习成本，点击即可完成核心操作（如重置游戏、调节速度）。

- 思考过程、操作记录展示清晰，易于理解，无歧义。

- 游戏流程引导明确，用户可快速了解当前游戏阶段及核心信息。

## 4.4 稳定性需求

- 游戏过程中无崩溃、无死循环，AI操作符合规则，不出现逻辑错误。

- 页面刷新后，可选择恢复当前游戏进度或重置游戏，数据不丢失（临时存储）。

# 5. 约束条件

- 技术约束：不涉及后端持久化存储（游戏数据仅临时存储，页面关闭后丢失），无需用户注册、登录功能。

- 规则约束：严格遵循标准狼人杀规则，不支持自定义规则（如添加特殊角色、修改胜负条件）。

- AI约束：AI决策逻辑基于预设角色规则结合动态配置的LLM模型生成，不涉及复杂机器学习训练，确保决策可预测、可理解；模型配置需支持灵活调整，不影响游戏核心流程运行。

- 界面约束：风格简洁、统一，以展示游戏流程和AI思考为主，不添加多余装饰，避免干扰用户观察。

# 6. 验收标准

1. 功能验收：所有详细功能需求均实现，AI机器人可自主完成狼人杀全流程，思考过程、操作行为符合要求。

2. 性能验收：页面加载、AI响应、流程切换无明显卡顿，符合性能需求。

3. 兼容性验收：在主流浏览器、不同电脑屏幕分辨率下，页面正常显示、操作正常。

4. 易用性验收：界面简洁、操作便捷，用户可快速上手，思考过程、游戏信息展示清晰。

5. 稳定性验收：连续运行3局游戏，无崩溃、无逻辑错误，数据临时存储正常。
