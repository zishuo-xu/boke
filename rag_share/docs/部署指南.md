# RAG 可视化演示 - 部署指南

## 环境要求

| 组件 | 版本要求 |
|------|----------|
| Python | 3.10+ |
| Node.js | 16+ |
| npm | 8+ |
| Docker | 20+ (可选) |
| Docker Compose | 2.0+ (可选) |

---

## 方式一：传统部署

### 1. 后端部署

```bash
# 克隆代码
git clone <your-repo-url>
cd rag_share/backend

# 创建虚拟环境（推荐）
python -m venv venv
source venv/bin/activate  # Linux/Mac
# venv\Scripts\activate     # Windows

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env  # 从 .env.example 复制
# 编辑 .env 填入你的 API Key
vim .env

# 启动服务
uvicorn main:app --host 0.0.0.0 --port 8000
```

### 2. 前端部署

```bash
cd rag_share/frontend

# 安装依赖
npm install

# 开发模式运行
npm run dev

# 或生产构建
npm run build
```

### 3. Nginx 配置（生产环境）

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /path/to/rag_share/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 方式二：Docker 部署（推荐）

### 1. 创建 Dockerfile

**backend/Dockerfile**
```dockerfile
FROM python:3.10-slim

WORKDIR /app

COPY backend/requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY backend/ .

EXPOSE 8000

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**frontend/Dockerfile**
```dockerfile
FROM node:18-alpine AS builder

WORKDIR /app
COPY frontend/package*.json .
RUN npm ci

COPY frontend/ .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY frontend/nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 3000
CMD ["nginx", "-g", "daemon off;"]
```

**frontend/nginx.conf**
```nginx
server {
    listen 3000;
    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
    }
}
```

### 2. 创建 docker-compose.yml

```yaml
version: '3.8'

services:
  backend:
    build:
      context: .
      dockerfile: backend/Dockerfile
    ports:
      - "8000:8000"
    environment:
      - EMBEDDING_API_KEY=${EMBEDDING_API_KEY}
      - EMBEDDING_MODEL=${EMBEDDING_MODEL:-doubao-embedding-vision-251215}
      - EMBEDDING_BASE_URL=${EMBEDDING_BASE_URL:-https://ark.cn-beijing.volces.com/api/v3}
      - LLM_API_KEY=${LLM_API_KEY}
      - LLM_MODEL=${LLM_MODEL:-MiniMax-M2.7}
      - LLM_BASE_URL=${LLM_BASE_URL:-https://newapi.hizui.cn/v1}
    restart: unless-stopped

  frontend:
    build:
      context: .
      dockerfile: frontend/Dockerfile
    ports:
      - "3000:3000"
    depends_on:
      - backend
    restart: unless-stopped
```

### 3. 创建 .env.example

```bash
# Embedding API (火山引擎 Doubao)
EMBEDDING_API_KEY=your_embedding_api_key
EMBEDDING_MODEL=doubao-embedding-vision-251215
EMBEDDING_BASE_URL=https://ark.cn-beijing.volces.com/api/v3

# LLM API (MiniMax)
LLM_API_KEY=your_llm_api_key
LLM_MODEL=MiniMax-M2.7
LLM_BASE_URL=https://newapi.hizui.cn/v1
```

### 4. 启动服务

```bash
# 复制环境变量配置
cp .env.example .env
# 编辑 .env 填入你的 API Key

# 启动所有服务
docker-compose up -d

# 查看状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

访问 `http://服务器IP` 即可。

## 常见问题

### 1. 端口被占用
```bash
# 查找占用端口的进程
lsof -i :8000
lsof -i :3000

# 杀死进程
kill -9 <PID>
```

### 2. Docker 构建失败
```bash
# 清理 Docker 缓存
docker-compose build --no-cache
```

### 3. API 调用失败
- 检查 `.env` 中的 API Key 是否正确
- 检查服务器网络是否能访问外部 API
- 查看后端日志：`docker-compose logs backend`

### 4. 前端静态资源加载失败
- 确保 nginx 配置正确
- 检查 `dist` 目录是否存在

## 生产环境优化

1. **使用 PM2 管理后端进程**
   ```bash
   npm install -g pm2
   pm2 start uvicorn --name "rag-backend" -- main:app --host 0.0.0.0 --port 8000
   ```

2. **配置 HTTPS**
   - 使用 Let's Encrypt 免费证书
   - 或使用 Nginx 反向代理配置 SSL

3. **配置防火墙**
   ```bash
   # 开放必要端口
   sudo firewall-cmd --permanent --add-port=3000/tcp
   sudo firewall-cmd --reload
   ```
