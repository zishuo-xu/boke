## API 设计

### 基础信息
- Base URL: `http://localhost:8000`
- CORS: 允许 `http://localhost:5173`

### 接口列表

#### 1. 健康检查
```
GET /api/health
Response: { "status": "ok" }
```

#### 2. RAG 流程处理
```
POST /api/rag/process
Content-Type: application/json

Request:
{
  "text": "原始文本内容",
  "query": "用户问题",
  "chunk_size": 50,
  "overlap": 10,
  "top_k": 3,
  "use_rerank": true,
  "chunking_strategy": "by_chars",  // "by_chars" | "by_sentence" | "by_paragraph"
  "use_pg": false  // 是否使用 PostgreSQL 混合检索
}

Response:
{
  "preprocessed_text": "预处理后的文本",
  "chunks": [
    {"id": "chunk_0", "text": "...", "length": 50},
    ...
  ],
  "vectors": [[0.1, -0.2, ...], ...],  // 2048维向量
  "query_vector": [0.1, -0.2, ...],    // 2048维向量
  "retrieval_results": [
    {"chunk_id": "chunk_0", "similarity": 0.85, "vector_score": 0.85, "fulltext_score": 0.6},
    ...
  ],
  "reranked_results": [
    {"chunk_id": "chunk_0", "original_rank": 1, "new_rank": 1, "similarity": 0.85},
    ...
  ],
  "prompt": "完整Prompt内容",
  "answer": "生成的答案",
  "storage_mode": "memory" | "pg",
  "search_mode": "vector" | "hybrid"
}
```

#### 3. 获取嵌入向量
```
POST /api/rag/embed
Content-Type: application/json

Request:
{

Response:

  "embeddings": [
    [0.1, -0.2, ...],  // 第一个文本的向量
    [0.3, 0.1, ...]    // 第二个文本的向量

#### 4. 获取步骤列表
```

## 数据模型

### Chunk
```typescript
interface Chunk {
  id: string       // "chunk_0", "chunk_1", ...
  text: string     // chunk 内容
  length: number   // 字符数
}
```

### RetrievalResult
```typescript
interface RetrievalResult {
  chunk_id: string  // chunk ID
  similarity: number // 融合分数（混合检索模式）或向量相似度（内存模式）
  vector_score?: number   // 向量语义相似度（仅 PostgreSQL 混合检索）
  fulltext_score?: number // 全文关键词匹配率（仅 PostgreSQL 混合检索）
}
```

### RerankedResult
```typescript
interface RerankedResult {
  chunk_id: string
  original_rank: number  // 原始排名
  new_rank: number      // 重排后排名
  similarity: number
}
```

### RagProcessResult
```typescript
interface RagProcessResult {
  preprocessed_text: string
  chunks: Chunk[]
  vectors: number[][]        // 每个chunk的向量
  query_vector: number[]     // query的向量
  retrieval_results: RetrievalResult[]
  reranked_results: RerankedResult[] | null
  prompt: string
  answer: string
}
```

## 核心算法

### 1. 预处理
- 去除多余空格
- 处理特殊空白字符（制表符、不间断空格等）
- 统一换行符
- 处理特殊引号
- 清理空行但保留段落分隔

### 2. 切片策略

#### 按字符切片 (`by_chars`)
- 中文字符逐字符切分
- 英文按空格分词
- 按 `chunk_size` 累积达到指定长度切分

#### 按句子切片 (`by_sentence`)
- 使用正则匹配句子分隔符（。！？!?）
- 句子完整保留，不截断
- 多句合并直到超过 `chunk_size`

#### 按段落切片 (`by_paragraph`)
- 按空行分割段落
- 段落完整保留，不截断
- 多段落合并直到超过 `chunk_size`

### 3. 向量化
- 调用外部 Doubao Embedding API
- 模型：`doubao-embedding-vision-251215`
- 向量维度：2048
- API endpoint: `/api/v3/embeddings/multimodal`
- 如果 API 调用失败，回退到模拟向量

### 4. 相似度计算
- 使用余弦相似度
- 公式：`cosine_similarity(A, B) = dot(A, B) / (||A|| * ||B||)`

### 5. 重排
- BM25 + 向量相似度融合
- BM25 权重：0.4（衡量关键词匹配程度）
- 向量相似度权重：0.6（衡量语义相似程度）
- 融合公式：`fused = 0.4 * norm_bm25 + 0.6 * norm_vector`
- 使用 2-gram 中文分词计算 BM25 分数

### 6. PostgreSQL 混合检索

#### 数据库配置
```python
PG_CONFIG = {
    "host": os.getenv("PG_HOST", "47.107.187.18"),
    "port": int(os.getenv("PG_PORT", "5432")),
    "database": os.getenv("PG_DATABASE", "rag"),
    "user": os.getenv("PG_USER", "rag_user"),
    "password": os.getenv("PG_PASSWORD", "..."),
}
```

#### 表结构
```sql
-- document_chunks: 文档片段表
CREATE TABLE document_chunks (
    id VARCHAR(100) PRIMARY KEY,
    text TEXT NOT NULL,
    length INTEGER NOT NULL,
    textsearch TSVECTOR,        -- 全文检索向量
    metadata JSONB DEFAULT '{}',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- chunk_vectors: 向量表
CREATE TABLE chunk_vectors (
    chunk_id VARCHAR(100) PRIMARY KEY,
    embedding VECTOR(1024),      -- pgvector 最多 2000 维，截断到 1024
    FOREIGN KEY (chunk_id) REFERENCES document_chunks(id) ON DELETE CASCADE
);

-- 全文检索索引
CREATE INDEX idx_document_chunks_textsearch ON document_chunks USING GIN (textsearch);
```

#### 两路召回

**第一路 - 向量语义检索**
- 使用 `pgvector` 的 `<=>` 操作符（余弦距离）
- 按向量距离排序，召回 top_k * 3 条
- 分数：`1 - cosine_distance`（归一化到 0-1）

**第二路 - 全文关键词检索**
- 提取查询关键词：中文 bigram + 英文单词
- 使用 `ILIKE` 模式匹配
- 分数 = 匹配到的关键词数 / 总关键词数

#### 分数融合
```python
fused = vector_weight * vector_score + (1 - vector_weight) * fulltext_score
# vector_weight = 0.6（可配置）
```

#### 存储限制
- pgvector HNSW/IVFFlat 索引最大维度 2000
- embedding 原始维度 2048，存储时截断到 1024
- 向量检索精度可能略有下降，演示够用

## 前端状态管理

### 全局状态 (App.tsx)
```typescript
interface AppState {
  currentStep: number      // 当前步骤 1-9
  isProcessing: boolean    // 是否正在处理
  result: RagProcessResult | null  // API 返回结果
}

// 输入配置
interface ProcessRequest {
  text: string
  query: string
  chunk_size: number
  overlap: number
  top_k: number
  use_rerank: boolean
  chunking_strategy: string
  use_pg?: boolean  // 是否使用 PostgreSQL 混合检索
}

// 本地编辑状态
const [editedChunks, setEditedChunks] = useState<Chunk[] | null>(null)
```

### 状态流转
1. 用户输入文本 → 点击"开始演示"
2. 调用 `/api/rag/process` 获取结果
3. 跳转到步骤2，显示预处理结果
4. 用户可编辑切片，编辑后 `editedChunks` 更新
5. 后续步骤使用 `editedChunks || result.chunks`
