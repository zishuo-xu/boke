## API 设计

### 基础信息
- Base URL: `http://localhost:8000`
- CORS: 允许 `http://localhost:5173`

### 接口列表

#### 1. 健康检查
```
GET /api/health
Response: { "status": "ok" }
```

#### 2. RAG 流程处理
```
POST /api/rag/process
Content-Type: application/json

Request:
{
  "text": "原始文本内容",
  "query": "用户问题",
  "chunk_size": 50,
  "overlap": 10,
  "top_k": 3,
  "use_rerank": true,
  "chunking_strategy": "by_chars"  // "by_chars" | "by_sentence" | "by_paragraph"
}

Response:
{
  "preprocessed_text": "预处理后的文本",
  "chunks": [
    {"id": "chunk_0", "text": "...", "length": 50},
    ...
  ],
  "vectors": [[0.1, -0.2, ...], ...],  // 2048维向量
  "query_vector": [0.1, -0.2, ...],    // 2048维向量
  "retrieval_results": [
    {"chunk_id": "chunk_0", "similarity": 0.85},
    ...
  ],
  "reranked_results": [
    {"chunk_id": "chunk_0", "original_rank": 1, "new_rank": 1, "similarity": 0.85},
    ...
  ],
  "prompt": "完整Prompt内容",
  "answer": "生成的答案"
}
```

#### 3. 获取嵌入向量
```
POST /api/rag/embed
Content-Type: application/json

Request:
{
  "texts": ["文本1", "文本2"]
}

Response:
{
  "embeddings": [
    [0.1, -0.2, ...],  // 第一个文本的向量
    [0.3, 0.1, ...]    // 第二个文本的向量
  ]
}
```

#### 4. 获取步骤列表
```
GET /api/rag/steps
Response:
{
  "steps": [
    {"id": 1, "name": "原始文本", "description": "..."},
    ...
  ]
}
```

## 数据模型

### Chunk
```typescript
interface Chunk {
  id: string       // "chunk_0", "chunk_1", ...
  text: string     // chunk 内容
  length: number   // 字符数
}
```

### RetrievalResult
```typescript
interface RetrievalResult {
  chunk_id: string  // chunk ID
  similarity: number // 相似度 0-1
}
```

### RerankedResult
```typescript
interface RerankedResult {
  chunk_id: string
  original_rank: number  // 原始排名
  new_rank: number      // 重排后排名
  similarity: number
}
```

### RagProcessResult
```typescript
interface RagProcessResult {
  preprocessed_text: string
  chunks: Chunk[]
  vectors: number[][]        // 每个chunk的向量
  query_vector: number[]     // query的向量
  retrieval_results: RetrievalResult[]
  reranked_results: RerankedResult[] | null
  prompt: string
  answer: string
}
```

## 核心算法

### 1. 预处理
- 去除多余空格
- 处理特殊空白字符（制表符、不间断空格等）
- 统一换行符
- 处理特殊引号
- 清理空行但保留段落分隔

### 2. 切片策略

#### 按字符切片 (`by_chars`)
- 中文字符逐字符切分
- 英文按空格分词
- 按 `chunk_size` 累积达到指定长度切分

#### 按句子切片 (`by_sentence`)
- 使用正则匹配句子分隔符（。！？!?）
- 句子完整保留，不截断
- 多句合并直到超过 `chunk_size`

#### 按段落切片 (`by_paragraph`)
- 按空行分割段落
- 段落完整保留，不截断
- 多段落合并直到超过 `chunk_size`

### 3. 向量化
- 调用外部 Doubao Embedding API
- 模型：`doubao-embedding-vision-251215`
- 向量维度：2048
- API endpoint: `/api/v3/embeddings/multimodal`
- 如果 API 调用失败，回退到模拟向量

### 4. 相似度计算
- 使用余弦相似度
- 公式：`cosine_similarity(A, B) = dot(A, B) / (||A|| * ||B||)`

### 5. 重排
- 模拟重排逻辑
- 30% 概率进行位置交换
- 交换范围为相邻元素

## 前端状态管理

### 全局状态 (App.tsx)
```typescript
interface AppState {
  currentStep: number      // 当前步骤 1-9
  isProcessing: boolean    // 是否正在处理
  result: RagProcessResult | null  // API 返回结果
}

// 输入配置
interface ProcessRequest {
  text: string
  query: string
  chunk_size: number
  overlap: number
  top_k: number
  use_rerank: boolean
  chunking_strategy: string
}

// 本地编辑状态
const [editedChunks, setEditedChunks] = useState<Chunk[] | null>(null)
```

### 状态流转
1. 用户输入文本 → 点击"开始演示"
2. 调用 `/api/rag/process` 获取结果
3. 跳转到步骤2，显示预处理结果
4. 用户可编辑切片，编辑后 `editedChunks` 更新
5. 后续步骤使用 `editedChunks || result.chunks`
