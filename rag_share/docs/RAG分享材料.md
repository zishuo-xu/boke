# RAG 技术分享

## RAG 概述

### 什么是 RAG？

RAG = Retrieval-Augmented Generation（检索增强生成）

RAG 是一种结合检索系统和生成模型的技术，让 AI 能够基于外部知识库生成更准确、更可靠的回答。

```
用户问题 → 检索系统 → 找到相关文档 → 组装Prompt → 大模型生成 → 最终答案
```

### 为什么需要 RAG？

| 问题 | 纯生成 | RAG |
|------|--------|-----|
| 知识时效性 | 依赖训练数据 | 可实时更新知识库 |
| 幻觉问题 | 容易编造答案 | 基于真实文档回答 |
| 可追溯性 | 无法验证 | 可引用具体来源 |
| 成本 | 低 | 需要维护检索系统 |

### RAG 工作流程

RAG 流程分为两个阶段：

**索引构建（离线）**
```
原始文档 → 预处理 → 切片 → 向量化 → 存入向量数据库
```

**检索生成（在线）**
```
用户问题 → 向量化Query → 相似度检索 → 重排 → 组装Prompt → 大模型生成
```

## 各步骤详解

---

### 步骤 1：原始文本

**是什么**

RAG 流程的起点，是即将进入处理流程的原始文档内容。可能来自知识库、文档库、数据库等。

**为什么**

原始文档通常存在以下问题，不能直接用于检索：
- 包含噪声（多余空格、特殊符号、HTML标签等）
- 结构不统一，格式杂乱
- 可能包含广告、水印、脚注等干扰信息

**怎么做**

准备高质量的文档数据：
- 确保内容完整、语义连贯
- 排除无关广告、水印等干扰信息
- 文档格式可以是 TXT、PDF、HTML、Markdown 等

**效果**

得到一份原始文档，为后续处理做准备。

---

### 步骤 2：预处理

**是什么**

对原始文本进行清洗和标准化的过程，包括去除噪声、统一格式、规范化文本结构。

**为什么**

- 去除多余空格、换行、特殊字符等噪声
- 统一编码格式，消除乱码隐患
- 规范化文本结构，便于后续切片
- 减少 embedding 时的干扰信息

**怎么做**

常见预处理操作：
```
1. 去除多余空白：多个空格合并为一个
2. 规范化换行：换行符统一处理
3. 清除特殊字符：移除无意义的符号
4. 统一标点：全角转半角等
```

**效果**

得到一份干净、规范的预处理文本，字符数可能略有减少但语义保持完整。

---

### 步骤 3：切片（Chunking）

**是什么**

切片（Chunking）是将长文档切分成多个较小的、语义相对独立的文本块（chunks）。

```
原始文档（10000字符）
        ↓
    切片处理
        ↓
chunk_0: "人工智能（AI）是计算机科学的一个分支..."
chunk_1: "机器学习是人工智能的一个子集..."
chunk_2: "深度学习是机器学习的一个重要分支..."
chunk_3: "大型语言模型（LLM）是..."
...
```

**为什么**

1. **模型上下文限制**：大模型有 token 数量限制（如 4K、8K、128K），无法一次性处理完整长文档

2. **提高检索精度**：文档太长会导致关键信息被稀释，切成小块后更容易精确匹配用户问题

3. **来源可追溯**：明确知道答案来自哪个 chunk，而不是整篇文档

4. **成本控制**：发送给模型的 context 更少，响应更快、成本更低

**怎么做**

**1. 选择切片策略：**

| 策略 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 按字符 | 固定字符数切分 | 简单、大小均匀 | 可能切断句子 |
| 按句子 | 按句号、问号等切分 | 语义完整 | 大小不均匀 |
| 按段落 | 按段落边界切分 | 语义连贯 | 可能包含噪声 |
| 递归切片 | 按层级尝试多种方式 | 平衡效果 | 复杂度高 |

**2. 按字符切片（示例）：**
```
原文：深度学习是机器学习的重要技术
chunk_size=6, overlap=2

chunk_0: 深度学习是机  (位置0-5)
chunk_1: 是机器学习的  (位置4-9)  ← 与上一块重叠2字符
chunk_2: 学习的重要技  (位置8-13)
```

**3. 按句子切片（示例）：**
```
原文：
"人工智能（AI）是计算机科学的一个重要分支，致力于开发智能系统。机器学习是人工智能的一个子集。深度学习是机器学习的一个重要分支。"

按句子切分后：
chunk_0: 人工智能（AI）是计算机科学的一个重要分支，致力于开发智能系统。
chunk_1: 机器学习是人工智能的一个子集。
chunk_2: 深度学习是机器学习的一个重要分支。
```

**4. 按段落切片（示例）：**
```
原文：
【段落1】人工智能（Artificial Intelligence，AI）是计算机科学的一个重要分支，致力于开发能够执行通常需要人类智能的任务的系统。

【段落2】机器学习是人工智能的一个子集，它使系统能够从数据中学习并改进性能。深度学习则是机器学习的一个分支，使用多层神经网络。

【段落3】大型语言模型（LLM）是深度学习在自然语言处理领域的应用，能够理解和生成人类语言。

按段落切分后：
chunk_0: 人工智能（Artificial Intelligence，AI）是计算机科学的一个重要分支，致力于开发能够执行通常需要人类智能的任务的系统。
chunk_1: 机器学习是人工智能的一个子集，它使系统能够从数据中学习并改进性能。深度学习则是机器学习的一个分支，使用多层神经网络。
chunk_2: 大型语言模型（LLM）是深度学习在自然语言处理领域的应用，能够理解和生成人类语言。
```

**5. 设置关键参数：**

| 参数 | 说明 | 建议值 | 注意事项 |
|------|------|--------|----------|
| chunk_size | 每个 chunk 的目标大小 | 300-500 字符 | 太小丢失上下文，太大降低精度 |
| overlap | 相邻 chunk 重叠字符数 | 10-20% 的 chunk_size | 太多增加冗余，太少丢失边界信息 |

**效果**

得到多个语义相对完整的 chunks，每个 chunk 可以独立检索。切片的质量直接影响后续检索和回答的效果。

### 步骤 4：向量化（Embedding）

**是什么**

向量化（Embedding）是将文本转换为数值向量的过程。在向量空间中，每个文本被表示为一个由数字组成的有序列表。

```
文本 "人工智能" → 向量 [0.12, -0.34, 0.56, 0.23, ..., -0.78]
文本 "机器学习" → 向量 [0.45, -0.12, 0.67, 0.89, ..., 0.34]
文本 "做菜方法" → 向量 [-0.23, 0.56, -0.34, 0.12, ..., 0.67]
```

向量中的每个数字称为"维度"，一个向量可以有几百到几千个维度。

**为什么**

1. **计算机只能处理数字**：文本语义无法被计算机直接理解，需要转换为数值形式

2. **捕捉语义关系**：
   - 语义相似的文本 → 向量距离更近
   - 语义无关的文本 → 向量距离更远

   ```
   示例（简化二维空间）：
                    ↑ "机器学习" (0.8, 0.6)
                    |
                    |  "深度学习" (0.7, 0.5)
                    |
   文本向量空间 ----|------------------------→
                    |  "人工智能" (0.5, 0.3)
                    |
                    |  "做菜方法" (-0.3, -0.5)
                    |
   ```

3. **支持数学运算**：可以在向量空间中进行相似度计算

**怎么做**

1. **选择 Embedding 模型**：将文本转换为向量

   常用模型及特点：

   | 模型 | 维度 | 特点 |
   |------|------|------|
   | OpenAI text-embedding-3-large | 3072 | 效果好，支持多语言 |
   | OpenAI text-embedding-3-small | 1536 | 效果好，轻量级 |
   | Sentence-BERT | 768 | 适合句子匹配 |
   | Doubao Embedding | 1024 | 中文效果好 |

2. **调用 API 获取向量**：

   ```python
   # 示例：调用 Doubao Embedding API
   response = requests.post(
       "https://ark.cn-beijing.volces.com/api/v3/embeddings",
       headers={"Authorization": f"Bearer {API_KEY}"},
       json={
           "model": "doubao-embedding-vision-251215",
           "input": [{"type": "text", "text": "机器学习是人工智能的重要分支"}]
       }
   )
   embedding = response.json()["data"]["embedding"]
   # 输出：[0.12, -0.34, 0.56, ..., 0.89] (1024维)
   ```

3. **向量存储**：将向量存入向量数据库

   ```
   Chunk ID    | 向量
   ------------|----------------------------------
   chunk_0    | [0.12, -0.34, 0.56, ..., 0.89]
   chunk_1    | [0.45, -0.12, 0.67, ..., 0.34]
   chunk_2    | [-0.23, 0.56, -0.34, ..., 0.67]
   ```

**效果**

- 每个 chunk 被映射为一个高维向量（如 1024 维）
- 语义相似的文本，向量在空间中的距离更近
- 向量可以用于快速相似度检索

### 步骤 5：用户提问

**是什么**

用户提出的问题或查询意图，是 RAG 检索的触发点。

**为什么**

- Query 决定了检索的方向和目标
- Query 本身也需要向量化才能参与相似度计算
- Query 与 chunks 在同一向量空间中进行匹配
- Query 的质量直接影响检索效果

**怎么做**

1. 用户输入自然语言问题
2. 系统对 Query 进行预处理（与文档预处理类似）
3. 将预处理后的 Query 发送给 Embedding 模型
4. 得到 Query 对应的向量表示

**效果**

得到 Query 的向量表示，准备进入检索流程。

---

### 步骤 6：相似度检索

**是什么**

相似度检索是在向量数据库中查找与 Query 向量最相似的 chunks，通过计算向量之间的距离或夹角来衡量相似程度。

```
用户问题 → Query向量化 → 向量数据库检索 → 找到最相似的 chunks

示例：
Query: "什么是机器学习？"
Query 向量: [0.12, -0.34, 0.56, 0.23, ..., -0.78]

数据库中的 chunks：
  chunk_0: "深度学习是..." → [0.15, -0.30, 0.60, ...]
  chunk_1: "Python编程..." → [-0.20, 0.45, -0.10, ...]
  chunk_2: "机器学习是..." → [0.18, -0.28, 0.55, ...] ← 与 Query 最相似
  chunk_3: "烹饪食谱..." → [-0.35, 0.20, -0.40, ...]
```

**为什么**

1. **语义匹配**：能捕捉深层语义相似性，不只是关键词匹配

   ```
   传统关键词匹配：
   用户问："如何学习AI？"
   文档包含："人工智能是..." → 可能匹配不到（没有"学习"）

   向量语义匹配：
   用户问："如何学习AI？" → Query向量
   文档："人工智能是..." → 向量接近 → 能匹配
   ```

2. **高效处理海量数据**：通过 ANN 算法可以在毫秒级从百万级向量中找到最相似的

3. **支持模糊匹配**：同义词、近义词也能匹配

**怎么做**

**1. 计算相似度：**

三种常用的相似度度量方法：

| 方法 | 公式 | 值范围 | 特点 |
|------|------|--------|------|
| 余弦相似度 | cos(θ) = (A·B) / (|A|×|B|) | [-1, 1] | 衡量方向差异，对长度不敏感 |
| 点积 | A·B = Σaᵢbᵢ | 无界 | 简单高效，对长度敏感 |
| 欧氏距离 | d = √Σ(aᵢ-bᵢ)² | [0, +∞) | 几何距离，越小越相似 |

```
余弦相似度示例（简化二维向量）：

向量 A: (1, 0)     → 方向向右
向量 B: (0.707, 0.707) → 45度方向

cos(θ) = (1×0.707 + 0×0.707) / (1×1) = 0.707
两个向量夹角45度，相似度约0.71
```

**2. 检索算法：**

| 算法 | 原理 | 速度 | 精度 | 适用场景 |
|------|------|------|------|----------|
| 暴力搜索 | 计算所有向量距离 | 慢 | 100% | 小数据量 |
| IVF | 聚类后搜索最近邻 | 中 | 高 | 中等数据量 |
| HNSW | 图索引快速导航 | 快 | 很高 | 大规模数据 |
| PQ | 向量量化压缩 | 最快 | 中 | 超大规模 |

**3. 召回 Top-K：**

```
原始 chunks（共100个）：

chunk_0: [0.12, -0.34, ...] → 相似度 0.45
chunk_1: [0.45, -0.12, ...] → 相似度 0.72
chunk_2: [0.18, -0.28, ...] → 相似度 0.95  ← 第1名
chunk_3: [-0.23, 0.56, ...] → 相似度 0.35
...
chunk_99: [0.08, -0.15, ...] → 相似度 0.28

按相似度降序排列：

Top-3 召回结果：
  第1名: chunk_2 (0.95)
  第2名: chunk_1 (0.72)
  第3名: chunk_0 (0.45)
```

**4. 实际检索流程：**

```
1. Query 向量化
   "什么是机器学习？" → [0.12, -0.34, 0.56, ...] (1024维)

2. 相似度计算
   Query向量与数据库中所有chunk向量逐一计算相似度

3. 排序筛选
   按相似度降序排列

4. 召回结果
   返回 Top-K（如K=3）个最相关的chunk及其相似度分数
```

**效果**

- 召回 Top-K 个最相关的 chunks（如 K=3，返回前3名）
- 每个结果附带相似度分数（0-1之间，越高越相关）
- K-后的 chunks 标记为"未命中"
- 检索结果供后续重排和答案生成使用

### 步骤 6.5：检索方式对比

**向量检索 vs 混合检索**

| | 向量检索 | 混合检索 |
|---|---|---|
| **组成** | 纯向量相似度匹配 | 向量检索 + 关键词检索 |
| **检索方式** | ANN 算法搜索最近邻 | ANN + BM25/TF-IDF |
| **匹配依据** | 语义相似度 | 语义相似度 + 关键词匹配 |
| **优点** | 语义理解强，同义词可匹配 | 既懂语义又抓关键词 |
| **缺点** | 可能漏掉精确关键词 | 复杂度高，需要融合算法 |

**向量检索**

```
用户查询："苹果最新手机"

Query embedding → [0.12, -0.34, 0.56, ...]
↓
ANN 搜索找最近邻
↓
返回语义相关结果：
  - "iPhone 15 Pro" (0.92)
  - "新款手机评测" (0.88)
  - "Apple发布会" (0.85)
```

**混合检索**

```
用户查询："苹果最新手机"

同时执行两条检索路径：

路径1 - 向量检索：
  Query embedding → 语义相似结果
  - "iPhone 15 Pro" (0.92)
  - "手机处理器" (0.80)

路径2 - 关键词检索（BM25）：
  匹配"苹果"+"手机"关键词
  - "苹果官网最新iPhone" (score: 15.6)
  - "手机推荐苹果还是安卓" (score: 12.3)

结果融合（RRF 算法）：
  综合两条路径的排名，计算最终得分
  输出融合后的 Top-K 结果
```

**融合算法示例（RRF）**

Reciprocal Rank Fusion：对不同检索路径的结果按排名融合

```
向量检索结果排名：        关键词检索结果排名：
1. iPhone 15 Pro    →   1. 苹果官网iPhone
2. 新款手机评测     →   2. 手机对比苹果安卓
3. Apple发布会     →   3. 苹果最新动态

RRF 融合计算（假设 K=60）：
  iPhone 15 Pro:     1/61 + 1/(1+2) = 0.016 + 0.333 = 0.349
  苹果官网iPhone:    1/(1+4) + 1/61 = 0.200 + 0.016 = 0.216
  新款手机评测:       1/62 + 1/(1+4) = 0.016 + 0.200 = 0.216

最终排序：iPhone 15 Pro > 苹果官网iPhone ≈ 新款手机评测
```

**何时使用混合检索**

| 场景 | 推荐方式 |
|------|----------|
| 通用问答、语义搜索 | 纯向量检索 |
| 需要精确关键词匹配 | 混合检索 |
| 文档包含专有名词/代码 | 混合检索 |
| 海量数据、快速响应 | 纯向量检索 |
| 高精度需求场景 | 混合检索 + 重排 |

### 步骤 7：重排（Rerank）

**是什么**

重排（Rerank）是在向量检索召回 Top-K 结果后，使用更精细的重排模型对结果进行二次排序的过程。

向量检索通过计算向量相似度找到相关文档，但这种方法的匹配能力有限。重排模型能更深入地理解 Query 和 Document 之间的语义关系，重新计算相关性分数并排序。

```
检索流程：
Query → 向量检索（快速筛选）→ Top-K 结果 → 重排模型（精细排序）→ 最终排序结果
```

**为什么**

1. **向量检索的局限**：
   - 只能捕捉文本的表层语义相似度
   - 无法理解深层的查询意图
   - 可能召回"字面相似但语义不相关"的文档

2. **表面相似≠真正相关**：

   ```
   用户问题："如何用 Python 写一个网页爬虫？"

   向量检索召回：
   chunk_A: "Python 是一种编程语言，广泛用于数据分析" (相似度 0.92)
   chunk_B: "爬虫是一种自动化获取网页数据的技术" (相似度 0.88)
   chunk_C: "JavaScript 常用于网页前端开发" (相似度 0.75)

   问题：chunk_A 字面包含 Python，但实际不回答用户问题
   ```

3. **重排模型的优势**：
   - 使用更大的模型（如 Cross-Encoder）进行精细匹配
   - 能理解查询意图和文档内容的深层关系
   - 综合考虑更多语义因素

**怎么做**

1. **两阶段检索架构**：

   ```
   第一阶段 - 向量检索（召回）：
   - 使用 ANN（近似最近邻）算法快速查找
   - 召回 Top-50 或 Top-100 个候选结果
   - 优点：速度快、支持海量数据

   第二阶段 - 重排（精排）：
   - 使用 Cross-Encoder 模型精细打分
   - 从候选结果中选出 Top-5 或 Top-10
   - 优点：精度高、效果更好
   ```

2. **常用重排模型**：

   | 模型 | 类型 | 特点 |
   |------|------|------|
   | BGE-Reranker | 开源 | 效果好，支持中文 |
   | Cohere Rerank | 商业 API | 效果好，易于集成 |
   | Jina Reranker | 商业/开源 | 多语言支持 |
   | BM25 + 规则 | 传统 | 可作为补充策略 |

3. **重排示例**：

   ```
   用户问题：什么是机器学习？

   向量检索初排结果：
   chunk_2: "机器学习是人工智能的一个分支" (0.95)
   chunk_0: "人工智能技术发展迅速" (0.87)
   chunk_3: "深度学习是机器学习的一个分支" (0.82)
   chunk_1: "编程语言 Python 入门教程" (0.78)

   重排模型重新评估后：
   chunk_2: "机器学习是人工智能的一个分支" (0.95 → 0.98)  ✓ 最相关
   chunk_3: "深度学习是机器学习的一个分支" (0.82 → 0.91)  ↑ 语义高度相关，上升
   chunk_0: "人工智能技术发展迅速" (0.87 → 0.72)  ↓ 相关但不够精准
   chunk_1: "编程语言 Python 入门教程" (0.78 → 0.35)  ↓ 包含干扰词，下降

   说明：chunk_3 虽然初排分数较低，但重排后上升，因为它直接解释"机器学习"
   ```

4. **什么时候需要重排**：

   - 检索效果不佳，Top-K 结果不够精准
   - 需要更精确的排序来提升答案质量
   - 对回答质量要求较高的场景
   - 注意：重排会增加延迟和成本，简单场景可跳过

**效果**

- 得到更精确的排序结果，相关性更高的文档排在前面
- 可能显著改变初始检索的排序
- 提高最终答案的准确性和质量
- 代价是增加计算延迟和资源消耗

### 步骤 8：Prompt 组装

**是什么**

将系统指令、检索到的相关文档（Context）和用户问题组合成完整的 Prompt，发送给大模型。

**为什么**

- 大模型需要明确的上下文才能准确回答
- Context 帮助模型理解问题背景
- 系统指令设定模型的角色和回答规则
- 组装方式直接影响回答质量

**怎么做**

典型 Prompt 结构：
```
【系统指令】
你是一个专业的AI助手。你的任务是基于提供的上下文来回答用户问题。
如果上下文中没有相关信息，请如实告知。

【上下文】
chunk_1: 人工智能（AI）是计算机科学的一个重要分支...
chunk_2: 机器学习是人工智能的一个子集...

【用户问题】
什么是机器学习？

【回答要求】
请基于上述上下文回答问题。
```

组装原则：
- Context 要精选，不是越多越好
- 明确指示模型基于上下文回答
- 如果上下文不足，让模型承认不知道

**效果**

得到一个完整的 Prompt，可以发送给大模型生成最终答案。

---

### 步骤 9：答案生成

**是什么**

大模型基于组装好的 Prompt 生成最终的回答。

**为什么**

- 模型综合上下文信息和问题生成答案
- 答案基于真实检索结果，更可靠
- 可以追溯具体的信息来源
- 比纯生成更不易产生幻觉

**怎么做**

1. 将组装好的 Prompt 发送给大模型（如 GPT-4、Claude、文心一言等）
2. 模型根据上下文生成回答
3. 返回最终答案给用户

```
示例：
问题：什么是机器学习？
答案：机器学习是人工智能的一个子集，它使系统能够从数据中学习
并改进性能，而无需明确编程。这项技术通过算法识别数据模式，
并利用这些模式进行预测和决策。
```

**效果**

得到基于真实文档生成的回答，答案可追溯、质量更高。

## 总结

RAG 的核心价值：

1. **知识时效性**：可实时更新知识库，无需重新训练模型
2. **减少幻觉**：答案基于真实文档，更可靠
3. **可追溯**：可以引用具体的信息来源
4. **成本效率**：比微调更低，适用于频繁更新的知识场景
