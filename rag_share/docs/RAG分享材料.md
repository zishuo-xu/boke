# RAG 技术分享

## 一、什么是 RAG？

**RAG = Retrieval-Augmented Generation（检索增强生成）**

RAG 是一种结合检索系统和生成模型的技术，让 AI 能够基于外部知识库生成更准确、更可靠的回答。

```
用户问题 → 检索系统 → 找到相关文档 → 组装Prompt → 大模型生成 → 最终答案
```

### 为什么需要 RAG？

| 问题 | 纯生成 | RAG |
|------|--------|-----|
| 知识时效性 | 依赖训练数据 | 可实时更新知识库 |
| 幻觉问题 | 容易编造答案 | 基于真实文档回答 |
| 可追溯性 | 无法验证 | 可引用具体来源 |
| 成本 | 低 | 需要维护检索系统 |

## 二、RAG 工作流程

RAG 流程分为 **索引构建** 和 **检索生成** 两个阶段：

### 阶段一：索引构建（离线）

```
原始文档 → 预处理 → 切片(Chunking) → 向量化 → 存入向量数据库
```

1. **预处理**：清洗文本（去噪声、标准化格式）
2. **切片**：将长文档切成小块（chunks）
3. **向量化**：使用Embedding模型将文本转为向量
4. **存储**：将向量存入向量数据库（如 Milvus、Pinecone）

### 阶段二：检索生成（在线）

```
用户问题 → 向量化Query → 相似度检索 → 重排(Rerank) → 组装Prompt → 大模型生成
```

1. **Query 向量化**：将用户问题转为向量
2. **相似度检索**：在向量库中查找最相关的 chunks
3. **重排**：对召回结果重新排序
4. **Prompt 组装**：将相关文档和问题组合成完整 prompt
5. **生成回答**：调用大模型生成答案

## 三、关键技术与参数

### 切片策略（Chunking）

| 策略 | 原理 | 优缺点 |
|------|------|--------|
| 按字符 | 固定字符数切分 | 简单，可能切断句子 |
| 按句子 | 按句号/换行切分 | 保持语义完整，大小不均 |
| 按段落 | 按段落边界切分 | 语义连贯性强 |

**关键参数：**
- `chunk_size`：每个 chunk 的目标大小
- `overlap`：相邻 chunk 之间的重叠字符数（保持上下文连贯）

### 向量化（Embedding）

将文本映射到高维向量空间：
- 语义相似的文本 → 向量距离更近
- 向量维度：通常 768-1536 维
- 常用模型：OpenAI Embedding、Sentence-BERT、Doubao Embedding

### 相似度计算

| 方法 | 公式 | 特点 |
|------|------|------|
| 余弦相似度 | cos(θ) = A·B/\|A\|\|B\| | 值范围 [-1, 1]，方向敏感 |
| 点积 | A·B = Σaᵢbᵢ | 值范围无界，效率高 |
| 欧氏距离 | √Σ(aᵢ-bᵢ)² | 几何直观 |

### 重排（Rerank）

向量检索的补充：
- 初次检索可能遗漏语义细节
- 重排模型能更精细地评估 Query-Doc 匹配度
- 可能会改变初始排序

## 四、切片策略对比

### 按字符切片
```
原文：大语言模型是深度学习在自然语言处理领域的应用
chunk_size=20, overlap=5

chunk_2: 在自然语言处理领
chunk_3: 处理领域的应用
```

### 按句子切片
```
chunk_0: 人工智能（AI）是计算机科学的一个分支
chunk_1: 机器学习是人工智能的一个子集
chunk_2: 深度学习是机器学习的一个分支
chunk_3: 大型语言模型（LLM）是深度学习的应用
```

### 按段落切片
```

         致力于开发能够执行通常需要人类智能的任务的系统...

chunk_1: 机器学习是人工智能的一个子集，它使系统能够从数据中学习...

chunk_2: 大型语言模型（LLM）是深度学习在自然语言处理领域的应用...
```

## 六、最佳实践建议

### 切片优化
- Chunk 大小通常 300-500 字符
- Overlap 建议 10-20% 的 chunk_size
- 保持语义完整，优先按句子/段落切分

### 检索优化
- Top K 不宜过大（通常 3-10）
- 根据实际效果调整重排策略
- 可以添加元数据过滤（如时间、类别）

### Prompt 工程
- Context 要精选，不是越多越好
- 明确指示模型基于上下文回答
- 必要时让模型承认不知道

### Embedding 模型选择
- 中文场景优先选择中文效果好的模型
- 根据文档类型选择合适的 embedding 服务
- 注意向量维度对存储和检索性能的影响

## 七、常见问题

**Q: RAG 和微调有什么区别？**
A: 微调是让模型适应特定任务/风格；RAG 是给模型提供外部知识。RAG 更适合知识频繁更新的场景。

**Q: 什么时候不需要 RAG？**
A: 如果任务不依赖外部知识、答案可以从模型知识中准确回答、或者实时性要求不高，可以不用 RAG。

**Q: 如何评估 RAG 效果？**
A: 可以从召回率（相关文档是否被召回）、答案准确率（答案是否基于正确文档）、响应时间等维度评估。
