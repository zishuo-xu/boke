# 日常学习博客系统项目文档

## 1. 项目概述
这是一个基于 Flask 的个人博客系统，支持前台阅读、后台内容管理、Markdown 编辑、版本回滚、图片上传与媒体库、用户评论。

## 2. 技术栈
- 后端：Flask
- ORM：Flask-SQLAlchemy / SQLAlchemy
- 认证：Flask-Login
- Markdown：Python-Markdown + bleach（XSS 清洗）
- 编辑器：Editor.md
- 图片处理：Pillow
- 数据库：SQLite（默认）

## 3. 快速启动
```bash
cd /Users/xuzishuo/ai-work/boke
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
flask --app run.py init-db
flask --app run.py create-admin
flask --app run.py run
```

访问地址：
- 前台首页：`http://127.0.0.1:5000`
- 登录页：`http://127.0.0.1:5000/login`
- 后台：`http://127.0.0.1:5000/admin`

默认管理员：
- 用户名：`admin`
- 密码：`admin123`

## 4. 权限模型
- 游客：可浏览文章、搜索、分类、标签
- 普通登录用户：在前台可发表评论
- 管理员：可进入后台，管理文章/分类/标签/媒体

管理员白名单在配置中：
- `app/__init__.py` -> `ADMIN_USERNAMES={"admin"}`

## 5. 核心功能

### 5.1 前台功能
- 首页文章列表 + 分页
- 文章详情（Markdown 渲染、目录、上一篇/下一篇）
- 搜索（标题/摘要/正文）
- 分类页、标签页、按分类/标签筛选
- 评论系统（登录即发，无审核）

### 5.2 后台功能
- 文章新建/编辑（Editor.md 双栏实时预览）
- 显式动作：`保存草稿` / `发布文章`
- 自动摘要
- 自动保存（仅编辑页）
- 版本历史、版本预览、一键回滚
- 草稿/发布/回收站筛选
- 删除移入回收站、恢复为草稿、彻底删除
- 分类管理 + 新建页快速新增分类
- 标签管理（新增/批量删除/合并）

### 5.3 媒体能力
- 编辑器上传图片（`jpg/jpeg/png/webp`）
- 粘贴图片自动上传并插入 Markdown 图片链接
- 服务端自动缩放与压缩
- 后台媒体库（查看、复制链接、删除）

## 6. 关键路由

### 6.1 认证
- `GET/POST /login`
- `GET/POST /register`
- `GET /logout`

### 6.2 前台
- `GET /`
- `GET /post/<post_id>`
- `POST /post/<post_id>/comments`
- `POST /comments/<comment_id>/delete`
- `GET /search`
- `GET /categories`
- `GET /category/<category_id>`
- `GET /tags`
- `GET /tag/<tag_id>`

### 6.3 后台
- `GET /admin`
- `GET /admin/posts`
- `GET/POST /admin/posts/new`
- `GET/POST /admin/posts/<post_id>/edit`
- `POST /admin/posts/<post_id>/delete`（移入回收站）
- `POST /admin/posts/<post_id>/restore`
- `POST /admin/posts/<post_id>/destroy`
- `POST /admin/posts/autosave`
- `POST /admin/posts/<post_id>/rollback/<version_id>`
- `GET /admin/posts/<post_id>/versions/<version_id>`（版本预览）
- `GET /admin/media`
- `POST /admin/media/upload`
- `POST /admin/media/<asset_id>/delete`
- `GET/POST /admin/categories`
- `POST /admin/categories/quick-add`
- `POST /admin/categories/<category_id>/delete`
- `GET/POST /admin/tags`
- `POST /admin/tags/delete`
- `POST /admin/tags/merge`

## 7. 数据模型
- `User`：用户账号
- `Category`：文章分类
- `Tag`：文章标签
- `Post`：文章（状态：草稿/已发布/回收站）
- `post_tag`：文章-标签关联
- `PostDraft`：编辑页自动保存草稿
- `PostVersion`：版本快照与回滚依据
- `MediaAsset`：媒体库资源
- `Comment`：文章评论

## 8. 媒体上传配置
在 `app/__init__.py`：
- `MEDIA_UPLOAD_SUBDIR="uploads/media"`
- `MEDIA_MAX_BYTES=5 * 1024 * 1024`
- `MEDIA_MAX_DIMENSION=1920`
- `MEDIA_JPEG_QUALITY=82`

## 9. 运维与排错

### 9.1 新增功能后 500（常见）
通常是新表未创建，执行：
```bash
flask --app run.py init-db
```

### 9.2 编辑器上传/粘贴图片无反应
- 确认已安装依赖：`pip install -r requirements.txt`
- 确认登录管理员账号
- 确认上传接口可访问：`/admin/media/upload`

### 9.3 后台权限问题
- 检查 `ADMIN_USERNAMES` 配置
- 确认登录账号在管理员白名单中

## 10. 当前已知限制
- 搜索仍是 SQL LIKE（未接入 Whoosh）
- 评论无审核/敏感词过滤
- 未接入对象存储（当前图片保存在本地 `app/static/uploads/`）
