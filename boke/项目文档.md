# 日常学习博客系统项目文档

## 1. 项目概述
这是一个基于 Flask 的个人博客系统，支持前台阅读、后台内容管理、Markdown 编辑、版本回滚、图片上传与媒体库、用户评论。

## 2. 技术栈
- 后端：Flask
- ORM：Flask-SQLAlchemy / SQLAlchemy
- 认证：Flask-Login
- Markdown：Python-Markdown + bleach（XSS 清洗）
- 编辑器：Editor.md
- 图片处理：Pillow
- 数据库：SQLite（默认）

## 3. 快速启动
```bash
cd /Users/xuzishuo/ai-work/boke
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
flask --app run.py init-db
flask --app run.py create-admin
flask --app run.py run
```

访问地址：
- 前台首页：`http://127.0.0.1:5000`
- 登录页：`http://127.0.0.1:5000/login`
- 后台：`http://127.0.0.1:5000/admin`

默认管理员：
- 用户名：`admin`
- 密码：`admin123`

## 4. 权限模型
- 游客：可浏览文章、搜索、分类、标签
- 普通登录用户：在前台可发表评论
- 管理员：可进入后台，管理文章/分类/标签/媒体

管理员白名单在配置中：
- `app/__init__.py` -> `ADMIN_USERNAMES={"admin"}`

## 5. 核心功能

### 5.1 前台功能
- 首页文章列表 + 分页
- 文章详情（Markdown 渲染、目录、上一篇/下一篇）
- 搜索（标题/摘要/正文）
- 分类页、标签页、按分类/标签筛选
- 评论系统（登录即发，无审核）

### 5.2 后台功能
- 文章新建/编辑（Editor.md 双栏实时预览）
- 显式动作：`保存草稿` / `发布文章`
- 自动摘要
- 自动保存（仅编辑页）
- 版本历史、版本预览、一键回滚
- 草稿/发布/回收站筛选
- 删除移入回收站、恢复为草稿、彻底删除
- 分类管理 + 新建页快速新增分类
- 标签管理（新增/批量删除/合并）

### 5.3 媒体能力
- 编辑器上传图片（`jpg/jpeg/png/webp`）
- 粘贴图片自动上传并插入 Markdown 图片链接
- 服务端自动缩放与压缩
- 后台媒体库（查看、复制链接、删除）

## 7. 数据模型
- `User`：用户账号
- `Category`：文章分类
- `Tag`：文章标签
- `Post`：文章（状态：草稿/已发布/回收站）
- `post_tag`：文章-标签关联
- `PostDraft`：编辑页自动保存草稿
- `PostVersion`：版本快照与回滚依据
- `MediaAsset`：媒体库资源
- `Comment`：文章评论

## 8. 媒体上传配置
在 `app/__init__.py`：
- `MEDIA_UPLOAD_SUBDIR="uploads/media"`
- `MEDIA_MAX_BYTES=5 * 1024 * 1024`
- `MEDIA_MAX_DIMENSION=1920`
- `MEDIA_JPEG_QUALITY=82`

## 9. 运维与排错

### 9.1 新增功能后 500（常见）
通常是新表未创建，执行：
```bash
flask --app run.py init-db
```

### 9.2 编辑器上传/粘贴图片无反应
- 确认已安装依赖：`pip install -r requirements.txt`
- 确认登录管理员账号
- 确认上传接口可访问：`/admin/media/upload`

### 9.3 后台权限问题
- 检查 `ADMIN_USERNAMES` 配置
- 确认登录账号在管理员白名单中

## 10. 当前已知限制
- 搜索仍是 SQL LIKE（未接入 Whoosh）
- 评论无审核/敏感词过滤
- 未接入对象存储（当前图片保存在本地 `app/static/uploads/`）

### 11.4 本次部署过程中的关键修正

1. Docker Hub 拉取慢：
- 现象：`docker compose up --build` 卡在 `load metadata for python:3.11-slim`
- 处理：配置 Docker registry mirrors（`/etc/docker/daemon.json`）并重启 Docker

```bash
mkdir -p /etc/docker
cat > /etc/docker/daemon.json <<'EOF'
{
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://docker.1ms.run",
    "https://hub.rat.dev"
  ]
}
EOF
systemctl daemon-reload
systemctl restart docker
```

2. `pip install` 失败（SQLAlchemy 拉取不到）：
- 现象：`No matching distribution found for sqlalchemy>=2.0.16`
- 原因：网络超时导致 PyPI 索引不完整
- 处理：Dockerfile 使用清华源安装依赖

```dockerfile
RUN pip install --no-cache-dir -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt
```

3. 容器启动成功验证：
- `docker compose ps` 显示 `Up`
- `curl -I http://127.0.0.1:5000` 返回 `HTTP/1.1 200 OK`

### 11.8 常见故障排查

1. 公网无法访问
- 检查阿里云安全组是否放行 `5000/TCP`
- 检查服务器防火墙策略
- 检查容器是否 `Up`

2. 容器一直重启
```bash
docker compose logs --tail=200
```
通常先看数据库初始化、端口占用、依赖安装是否失败。

3. 端口冲突
- 若宿主机 `5000` 被占用，改 `docker-compose.yml` 端口映射，例如：
`"5001:5000"`

4. 数据丢失风险
- 当前使用 SQLite + 挂载目录
- 请定期备份：
  - `/root/boke/boke/instance/`
  - `/root/boke/boke/app/static/uploads/`

### 11.9 生产建议（下一步）
- 使用 `Gunicorn + Nginx` 替换 Flask 开发服务器
- 通过 80/443 暴露服务并配置 HTTPS
- 使用独立数据库（MySQL/PostgreSQL）
- 增加定时备份与监控告警
