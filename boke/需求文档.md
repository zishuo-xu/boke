# 日常学习博客分享系统需求文档
> 版本：V1.0  
> 更新日期：2026-03-04  
> 核心技术栈：Python后端 + Markdown内容支持  
> 项目定位：个人学习笔记沉淀、技术博客分享Web平台

---

## 1. 项目概述
### 1.1 项目目标
搭建一个轻量、易用的个人学习博客Web平台，基于Python技术栈实现后端能力，核心支持Markdown格式的内容创作、渲染与管理，满足个人日常学习笔记、技术总结、知识沉淀的线上分享需求。

### 1.2 目标用户
| 用户角色 | 权限范围 | 核心使用场景 |
|----------|----------|--------------|
| 管理员（博主） | 全量权限，含内容创作、编辑、删除、分类标签管理、系统配置 | 撰写学习笔记、发布技术博客、管理内容分类、维护平台内容 |
| 普通访客 | 只读权限，仅可浏览、搜索、筛选博客内容 | 查阅学习笔记、浏览技术文章、通过分类/标签检索相关内容 |

### 1.3 项目边界
- 核心范围：Markdown博客的全生命周期管理、内容展示与检索、基础权限控制
- 暂不纳入：多用户注册、评论互动、点赞收藏、付费阅读等社交化功能（预留扩展接口）

## 2. 功能需求
### 2.1 前台用户端功能
#### 2.1.1 首页展示模块
1.  博客文章列表展示，单条文章卡片包含：文章标题、摘要、发布时间、所属分类、关联标签
2.  列表分页功能，支持自定义每页展示数量
3.  顶部导航栏，包含：首页、分类入口、标签入口、搜索框、后台管理登录入口
4.  侧边栏辅助模块：最新文章推荐、标签云、分类统计

#### 2.1.2 文章详情模块
1.  Markdown格式内容完整渲染，支持代码块高亮、表格、图片、列表、标题层级等全语法支持
2.  文章元信息展示：发布时间、更新时间、所属分类、关联标签
3.  自动生成文章目录导航，支持锚点快速跳转
4.  文章末尾上一篇/下一篇快速跳转入口
5.  XSS攻击防护，对Markdown渲染内容进行恶意代码过滤

#### 2.1.3 内容检索模块
1.  全文检索功能，支持按文章标题、摘要、正文内容关键词搜索
2.  搜索结果高亮展示匹配关键词
3.  搜索结果分页展示，匹配度优先排序

#### 2.1.4 分类与标签模块
1.  分类列表页，展示全部分类及对应文章数量，点击可筛选该分类下的所有文章
2.  标签云页面，展示全部标签，支持按标签热度差异化展示
3.  点击标签可筛选该标签关联的所有文章
4.  分类/标签筛选结果支持分页展示

### 2.2 后台管理端功能
#### 2.2.1 身份认证模块
1.  管理员账号登录功能，支持账号密码校验
2.  登录状态Session管理，超时自动退出
3.  密码加密存储，禁止明文入库
4.  登出功能，清除登录状态

#### 2.2.2 文章管理模块
1.  文章新增功能：集成可视化Markdown编辑器，支持双栏编辑+实时预览
2.  文章编辑功能：支持修改文章标题、正文、分类、标签，保留修改记录
3.  文章删除功能：支持单篇/批量删除，二次确认机制
4.  文章状态管理：支持草稿/发布两种状态，草稿仅管理员可见
5.  自动生成文章摘要，支持手动修改调整

#### 2.2.3 分类与标签管理模块
1.  分类管理：支持分类的新增、编辑、删除、排序功能
2.  标签管理：支持标签的新增、批量删除、合并功能
3.  关联管理：支持文章与分类一对一绑定、与标签多对多绑定

#### 2.2.4 数据统计模块
1.  基础数据看板：文章总数、分类总数、标签总数统计
2.  文章发布时间分布统计
3.  分类/标签文章数量排行统计

### 2.3 后端核心接口能力
| 接口模块 | 核心接口能力 |
|----------|--------------|
| 认证接口 | 登录校验、登录状态查询、登出接口 |
| 文章接口 | 文章增删改查、草稿保存、状态更新、详情查询接口 |
| 分类接口 | 分类增删改查、分类下文章查询接口 |
| 标签接口 | 标签增删改查、标签关联文章查询接口 |
| 搜索接口 | 全文检索、关键词匹配、结果返回接口 |
| 静态资源接口 | 图片上传、Markdown附件存储与访问接口 |

## 3. 非功能需求
### 3.1 性能需求
1.  页面首屏加载时间≤2s，静态资源按需加载
2.  文章内容检索响应时间≤500ms
3.  Markdown内容渲染时间≤300ms
4.  单接口最大并发支持≥100QPS，满足个人博客访问量级

### 3.2 安全性需求
1.  防SQL注入：所有数据库操作采用ORM预编译处理，禁止拼接SQL语句
2.  防XSS攻击：Markdown渲染前进行恶意标签与脚本过滤
3.  权限隔离：后台管理接口全量鉴权，未登录用户禁止访问
4.  密码安全：采用不可逆加密算法存储密码，禁止明文传输与存储
5.  上传安全：限制图片上传格式与大小，禁止恶意文件上传

### 3.3 兼容性需求
1.  浏览器兼容：支持Chrome、Firefox、Safari、Edge等主流浏览器最新2个版本
2.  设备兼容：响应式布局，支持PC端、平板端正常浏览与操作
3.  渲染兼容：支持主流Markdown通用语法，兼容GFM语法规范

### 3.4 可扩展性与可维护性
1.  代码分层清晰，遵循MVC架构模式，业务逻辑与数据操作解耦
2.  Python代码严格遵循PEP 8编码规范，关键逻辑补充完整注释
3.  预留功能扩展接口，支持后续新增评论、RSS订阅、多用户等功能
4.  支持数据库平滑迁移，不影响线上存量数据

## 4. 技术选型规范
| 技术层级 | 选型方案 | 选型说明 |
|----------|----------|----------|
| 后端核心框架 | Flask | Python轻量级Web框架，灵活度高，适配个人博客轻量场景，生态完善 |
| 数据库 | 开发环境：SQLite<br>生产环境：PostgreSQL | SQLite无需额外部署，适配本地开发调试；PostgreSQL适配生产环境，支持全文检索、高并发稳定运行 |
| ORM框架 | SQLAlchemy + Flask-SQLAlchemy | Python主流ORM工具，简化数据库操作，支持数据迁移，适配多数据库切换 |
| Markdown处理 | Python-Markdown + Pygments | 后端完成Markdown解析渲染，Pygments实现代码块语法高亮，降低前端渲染压力 |
| 全文检索引擎 | Whoosh | 纯Python实现的轻量级全文检索引擎，无需额外部署服务，完美适配Flask技术栈 |
| 前端模板引擎 | Jinja2 | Flask原生配套模板引擎，实现服务端渲染，提升SEO友好度与首屏加载速度 |
| 前端UI框架 | Bootstrap 5 | 响应式UI框架，快速搭建兼容多设备的页面，降低前端开发成本 |
| Markdown编辑器 | Editor.md | 开源Markdown编辑器，支持双栏实时预览、GFM语法、代码高亮、图片上传，适配博客创作场景 |
| 身份认证 | Flask-Login | Flask生态成熟的登录认证组件，快速实现登录状态管理、权限鉴权 |

## 5. 数据结构设计
### 5.1 核心数据表设计
#### 5.1.1 用户表（user）
| 字段名 | 数据类型 | 约束 | 字段说明 |
|--------|----------|------|----------|
| id | INT | 主键、自增、非空 | 用户唯一ID |
| username | VARCHAR(50) | 非空、唯一 | 管理员登录账号 |
| password_hash | VARCHAR(255) | 非空 | 加密后的密码 |
| created_at | DATETIME | 非空 | 账号创建时间 |
| updated_at | DATETIME | 非空 | 账号信息更新时间 |

#### 5.1.2 分类表（category）
| 字段名 | 数据类型 | 约束 | 字段说明 |
|--------|----------|------|----------|
| id | INT | 主键、自增、非空 | 分类唯一ID |
| name | VARCHAR(30) | 非空、唯一 | 分类名称 |
| sort | INT | 默认0 | 分类排序权重，数字越小越靠前 |
| created_at | DATETIME | 非空 | 分类创建时间 |
| updated_at | DATETIME | 非空 | 分类更新时间 |

#### 5.1.3 标签表（tag）
| 字段名 | 数据类型 | 约束 | 字段说明 |
|--------|----------|------|----------|
| id | INT | 主键、自增、非空 | 标签唯一ID |
| name | VARCHAR(30) | 非空、唯一 | 标签名称 |
| created_at | DATETIME | 非空 | 标签创建时间 |
| updated_at | DATETIME | 非空 | 标签更新时间 |

#### 5.1.4 文章表（post）
| 字段名 | 数据类型 | 约束 | 字段说明 |
|--------|----------|------|----------|
| id | INT | 主键、自增、非空 | 文章唯一ID |
| title | VARCHAR(100) | 非空 | 文章标题 |
| content | TEXT | 非空 | Markdown格式的文章正文源码 |
| summary | VARCHAR(500) | 非空 | 文章摘要 |
| status | TINYINT | 非空、默认0 | 文章状态：0-草稿，1-已发布 |
| category_id | INT | 外键、关联category.id | 所属分类ID |
| view_count | INT | 默认0 | 文章阅读次数 |
| created_at | DATETIME | 非空 | 文章创建时间 |
| updated_at | DATETIME | 非空 | 文章最后更新时间 |

#### 5.1.5 文章-标签关联表（post_tag）
| 字段名 | 数据类型 | 约束 | 字段说明 |
|--------|----------|------|----------|
| post_id | INT | 主键、外键、关联post.id、非空 | 关联文章ID |
| tag_id | INT | 主键、外键、关联tag.id、非空 | 关联标签ID |

### 5.2 字段通用规范
1.  所有表必须包含创建时间`created_at`与更新时间`updated_at`字段
2.  所有主键统一命名为`id`，采用自增INT类型
3.  字符串类型字段必须设置合理长度，禁止滥用TEXT类型
4.  外键字段必须建立索引，提升查询效率
5.  所有非空字段必须设置默认值，避免空值异常

## 6. 界面设计规范
### 6.1 核心页面布局规范
1.  **首页**
    - 顶部固定导航栏，统一全站入口
    - 主体区域采用左右分栏：左侧70%宽度为文章列表，右侧30%宽度为侧边栏辅助模块
    - 底部设置分页控件与版权信息

2.  **文章详情页**
    - 顶部保留统一导航栏
    - 主体区域采用左右分栏：左侧20%宽度为固定文章目录导航，右侧80%宽度为文章正文内容
    - 文章末尾展示分类、标签信息，以及上一篇/下一篇跳转入口
    - 底部保留统一版权信息

3.  **后台管理页**
    - 左侧固定侧边栏，包含：文章管理、分类管理、标签管理、数据统计、退出登录入口
    - 右侧主体区域为对应功能的操作面板
    - 文章编辑页采用双栏布局：左侧50%为Markdown编辑区，右侧50%为实时预览区

### 6.2 视觉规范
1.  整体风格采用极简轻量化设计，以内容展示为核心，避免过度装饰
2.  主色调采用低饱和度色系，适配长时间阅读场景，降低视觉疲劳
3.  代码块采用深色主题，与正文内容形成视觉区分，提升可读性
4.  字体层级清晰：标题、正文、辅助信息采用差异化字号与字重，保证阅读流畅度

## 8. 验收标准
1.  功能完整性：需求文档中所有功能点全部实现，无遗漏、无阻塞性BUG
2.  核心能力达标：Markdown格式内容可正常编辑、保存、渲染，全语法支持无异常
3.  性能达标：页面加载、接口响应、内容检索均满足3.1章节的性能需求
4.  安全性达标：无SQL注入、XSS、越权访问等安全漏洞
5.  兼容性达标：主流浏览器、多设备均可正常访问与操作，无布局错乱、功能异常
